About Taste Test

Taste Test was created with the intent to reveal to users what

their taste in music looks like in a graphical matter and present it in

a customized playlist that will allow them to both see and hear their

unique take on what constitutes beauty in the musical world. Originally,

the heart of the application was going to be a neural network training

algorithm that allowed us to leverage each individual user's Spotify history

into useful training data. This data would be fed into our algorithm which would

lead to a unique neural network modelled after the user's taste in music. Using

this model, we could conceivalby run any given track on Spotify through it in order

to determine how much the user would enjoy said track. Unfortunately, this plan was

ultimately abandoned as the required amount of data needed to train each model was far

beyond our reach.

With this difficult decision, the Taste Team then pivoted towards a more genreal approach

in which a large, intricate neural network would instead be trained against a corpus

of Spotify's own music database in order to teach it how to analyze any track and determine

its genre composition. The neural network itself is of the forward-feeding type, which

which means the network lacks any sort of memory, solely relying on a type of linear

regression and integral weight randomization to create a learned model.

In essence, this model allows us to analyze any Spotify track and determine its similarity

to a set of given generes. With this information, we can simply calculate a user's genre

preference composition with a realtively straightforward scoring algorithm and then

run a corpus of tracks through the trained neural network in order to find the

most optimal set of tracks for a user.
